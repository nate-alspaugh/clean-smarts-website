## Design Context

### Users
**Audience:** Owners, operators, and managers of commercial cleaning and janitorial businesses. 
**Context:** They are evaluating operational software that centralizes tracking, scheduling, payroll, and tasks. They are often price-conscious but looking for robust, scalable solutions.
**Job to be Done:** Eliminate redundant software costs by consolidating into a single platform that handles the entire operational lifecycle (from time tracking to inspections).

### Brand Personality
Professional, Robust, Premium.
The brand must evoke confidence and scale—demonstrating that the software is reliable enough to handle a massive distributed workforce.

### Aesthetic Direction
**Visual Tone:** Strict, high-end premium.
**Theme:** Grayscale-based monochrome palette, heavily reliant on deep slate/off-blacks and stark whites. Bright generic SaaS colors (like standard emeralds and blues) should be minimized or entirely removed in favor of using typography, spacing, and subtle grays for visual hierarchy.
**Stack Constraints:** STRICTLY use **Base UI** (@base-ui/react) for structural/interactive primitives, and **Tailwind CSS** for all styling. No generic or custom raw HTML forms/toggles where a Base UI component exists.

### Design Principles
1. **Base-First Construction:** Always wrap functional UI (buttons, toggles, tabs, dialogues) in Base UI primitives for built-in accessibility, state management, and keyboard navigation.
2. **Premium Restraint:** Use typography weight, fluid spacing (`clamp`), and negative space—not color patches or borders—to define sections and hierarchy. 
3. **No AI Slop:** Strictly avoid unprompted colored drop shadows, glassmorphism, gradient text, or overly round, bubbly borders.
4. **Distilled Clarity:** Never repeat information. Use progressive disclosure to keep pages highly scannable and minimize cognitive load.
